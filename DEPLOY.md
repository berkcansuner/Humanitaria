# Deploy — Render (free tier, Docker)

One Docker web service runs the FastAPI API and serves the built Vue SPA (same
origin, so no CORS in the browser). Config lives in `render.yaml` + `Dockerfile`.

## Steps

1. **Render → New → Blueprint** and connect the GitHub repo
   `berkcansuner/reliefweb-rag`. Render reads `render.yaml` and creates the
   `reliefweb-rag` web service.

2. **Set the secret env vars** when prompted (these are `sync:false`, never in
   the repo):
   - `GEMINI_API_KEY`
   - `PINECONE_API_KEY`
   - `RELIEFWEB_APPNAME`
   - `GOOGLE_CLIENT_ID`
   - `GOOGLE_CLIENT_SECRET`

   `AUTH_SESSION_SECRET` is generated by Render automatically. The model/index
   settings use code defaults (index `reliefweb-docs`, default namespace,
   `gemini-3.5-flash`, embedding 3072) — set them only if yours differ.

3. **Deploy.** First build is a Docker build (frontend `npm ci`+build, then pip
   install) → ~5–10 min. Watch the build logs.

4. **Note the live URL.** If it is exactly `https://reliefweb-rag.onrender.com`,
   you're set. If Render appended a suffix (name taken), update these env vars in
   the dashboard to the real URL and redeploy:
   `FRONTEND_URL`, `GOOGLE_REDIRECT_URI` (`<url>/auth/google/callback`), `CORS_ORIGINS`.

5. **Google Cloud Console** → APIs & Services → Credentials → your OAuth client →
   add to **Authorized redirect URIs**:
   `https://<your-url>/auth/google/callback`
   (Consent screen still in "testing"? add the tester's email under Test users.)

6. **Open the URL** → Sign up (email/password) or "Continue with Google" → chat.

## Free-tier caveats

- **Data is not persistent.** No disk on free tier → `conversations.db` (users +
  conversations) resets on every deploy and on wake-from-sleep. Re-signup after a
  reset. For persistence: upgrade to a paid instance + disk, or migrate to Postgres.
- **Cold start.** Spins down after ~15 min idle; the next request takes ~30–60 s.
- **Pinecone write quota is full** → ingestion is disabled (`INGEST_SCHEDULE_HOURS=0`).
  Chat (reads) works against the existing index.
- **Gemini 503 ("high demand")** is transient/external; the chat route fast-fails
  with a "model busy" message instead of hanging.
